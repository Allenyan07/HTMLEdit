# 原型标注工具 — 产品需求文档 (PRD)

> **版本**: v2.3 | **状态**: 已交付 | **上次更新**: 2026-06-01
> **技术栈**: 纯原生 JavaScript，零依赖，单文件注入

---

## 一、产品概述

### 1.1 背景

原型评审和交付过程中，产品经理、设计师、开发之间需要对原型页面上的具体位置提出修改意见。传统的沟通方式（口头描述、截图+红线标注、表格汇总）存在以下问题：

- **描述不精确**："把那个按钮往右移一点" — 具体是哪？移多少？
- **上下文丢失**：单独截图无法反应完整的页面布局关系
- **归因困难**：多轮评审后难以追溯每条意见的提出时间和对应版本
- **效率低下**：沟通成本高，一个文案修改可能要来回确认多次

### 1.2 目标

打造一个**零侵入、可注入任意原型页面**的浏览器端标注工具，满足以下核心诉求：

1. 直接在原型页面上标注，所见即所得
2. 标注内容结构化，包含精确的 DOM 定位信息
3. 支持导出，方便开发直接基于标注定位修改
4. 零依赖，不修改页面原有代码

### 1.3 用户画像

| 角色 | 使用场景 | 核心诉求 |
|------|---------|---------|
| 产品经理 | 评审原型后标注修改意见 | 快速标注、分类清晰、导出给开发 |
| 设计师 | 审核 UI 还原度 | 精确标注间距/颜色/字体等视觉问题 |
| 开发工程师 | 接收修改工单 | 打开标注导出文件即可定位到对应元素 |

### 2.2 标注类型

| 类型 | 触发方式 | 数据字段 | 适用场景 |
|------|---------|---------|---------|
| 文本修改（✏️） | Shift+Click 元素 → 填写"修改为" | oldText, newText, note | 文案修改、链接文本替换等 |
| 修改说明（💬） | Shift+Click 元素 → 填写"修改说明" | note | 颜色调整、位置移动、交互行为变更 |
| 区域标注（📐） | Shift+拖拽框选区域 → 填写说明 | note, area{x,y,w,h} | 新增组件、删除区域、整体布局调整 |

### 2.3 导出格式

每条标注导出为结构化文本，按页面分组，包含图标字段、选择器、HTML上下文：

```
=== 原型标注导出（按页面分组） ===
日期: 2026/6/1 14:00:00
总计: 3 条

══════ 『购买上限控制 - 创建规则』 ══════（2 条）

  【A01】
  📄 原本内容：请输入审批备注说明...
  ✏️ 修改后：暑期活动矿泉水限购
  💬 修改说明：与活动名称保持一致
  选择器：div.form-body > div.form-row:nth-of-type(2) > textarea
  上下文：<div class="form-row"><label>备注</label><textarea data-target="">...</textarea></div>

  【A02】
  📐 区域标注
  💬 修改说明：这里新增一个数据看板组件
  区域坐标：left:160, top:240, width:400, height:300

══════ 『购买上限控制 - 审批记录』 ══════（1 条）

  【A03】
  📄 原本内容：暂存
  ✏️ 修改后：待审核
  💬 修改说明：状态标签颜色改为橙色
  选择器：table tbody tr:nth-of-type(1) td:nth-of-type(7) .status-tag
  上下文：<span class="status-tag status-tag-pending" data-target="">已提交</span>
=== 导出结束 ===
```

**字段说明：**

| 图标 | 字段 | 说明 |
|------|------|------|
| 📄 | 原本内容 | 元素的原始文本（元素标注） |
| ✏️ | 修改后 | 用户期望的新文本 |
| 💬 | 修改说明 | 颜色/位置/交互等非文本修改描述 |
| 📐 | 区域标注 | 框选区域的坐标和尺寸 |
| 💬 | 选择器 + 上下文 | CSS选择器定位代码行，HTML上下文确认唯一性 |

## 三、交互逻辑详细说明

### 3.1 核心流程

```
┌─────────────┐    点击 📌     ┌──────────────────┐
│  未激活状态  │ ─────────────→ │  标注模式已激活   │
│  📌 灰色    │               │  📌 蓝色 + 面板展开│
│  无面板     │               │  页面右推 340px    │
└─────────────┘               └──────────────────┘
       ↑                              │
       │        点击 📌               │
       └──────────────────────────────┘
             退出模式，全部收起
```

**声明确认**：标注模式采用"📌 按一下进入，再按一下退出"的双态开关设计。不存在面板收起但模式保留的中间状态。

### 3.2 标注操作

#### 元素标注（Shift+Click）

1. **按住 Shift** → 鼠标指针无变化（但底层已进入标注准备状态）
2. **点击任意页面元素**（排除工具自身 UI）→
   - 元素高亮（蓝色虚线框）
   - 弹出标注卡片（智能定位，防止溢出视口）
3. **填写内容**：
   - 「当前文本」灰色只读区域（自动抓取元素的 textContent/placeholder/value）
   - 「修改为」输入框 — 填了此项则为「文本修改」类型
   - 「修改说明」文本域 — 填了此项则为「修改说明」类型
   - 可同时填写两个字段，标注同时包含文本修改+说明
4. **确认**：点「确认」按钮或按 Enter 键
   - 校验：至少填写一个字段
   - 成功后：徽章出现在元素右上角，面板新增一条，Toast 提示
5. **取消**：点「取消」或按 Esc 键

#### 区域标注（Shift+拖拽）

1. **按住 Shift** → **鼠标按下并拖拽** → 出现蓝色虚线选择框
2. **松开鼠标** → 选择框固定，弹出区域标注卡片
3. **填写说明**：必须填写修改说明
4. **确认**：生成区域标注（无选择器，只有坐标和尺寸）
5. **编辑限制**：区域标注不支持从面板点击编辑（无法通过选择器找到对应元素），提示用户在页面上重新框选

#### 已有标注的编辑

- **徽章点击**：点击页面上标注编号徽章 → 自动定位、高亮元素、弹出编辑卡片（预填已有内容）
- **面板点击**：点击面板列表中的条目 → 滚动定位到页面元素、高亮、弹出编辑卡片
- **编辑保存**：修改后点击确认 → 覆盖已有标注数据

### 3.3 面板交互

| 要素 | 说明 |
|------|------|
| 面板位置 | 固定右侧，与页面内容并排 |
| 面板宽度 | 340px，页面内容自动右推同样宽度 |
| 面板头部 | 左「📋 标注列表」标题 / 右「📤 导出」「🗑 清空」 |
| 面板分组 | 按页面分组展示，每组有蓝色表头（📄 页面名 + 条数） |
| 面板条目 | 编号标签 + 内容区 + 删除按钮 |
| 条目内容 | 「✏️ 文本修改」+「💬 修改说明」分拆为独立div，各最多2行省略 |
| 条目点击 | 有选择器的条目可点击，跳转编辑 |
| 面板关闭 | 退出标注模式时自动隐藏 |

### 3.4 键盘快捷键

| 快捷键 | 作用域 | 功能 |
|--------|--------|------|
| Esc | 卡片打开时 | 关闭标注卡片，移除高亮 |
| Enter | 卡片打开时 | 确认提交当前标注 |
| Ctrl+Shift+E | 标注模式激活 | 弹出导出窗口 |
| Shift+Click | 标注模式激活 | 标注页面元素 |
| Shift+拖拽 | 标注模式激活 | 框选区域标注 |

### 3.5 页面布局变化

| 状态 | 页面内容 | 面板 | 📌 按钮位置 |
|------|---------|------|------------|
| 未激活 | 正常全宽 | 隐藏（display:none） | 右下角 24px |
| 激活展开 | padding-right: 340px | 固定右侧 340px | 右下角 364px（与内容区右边缘对齐） |

所有布局变化带 0.25s ease 过渡动画。

## 四、数据模型

### 4.1 标注数据结构

```typescript
interface Annotation {
  id: string;           // 编号 "A01", "A02" ...
  oldText?: string;     // 元素原始文本（文本修改标注）
  newText?: string;     // 修改后的文本（文本修改标注）
  note?: string;        // 修改说明
  selector?: string;    // CSS 选择器路径（元素标注）
  context?: string;     // 父元素 HTML 上下文（取前 600 字符）
  area?: Area;          // 区域坐标（区域标注）
  page: string;         // 页面文件名
  status: string;       // 状态 "pending"（预留扩展）
}

interface Area {
  left: number;
  top: number;
  width: number;
  height: number;
}
```

### 4.2 持久化存储

```
localStorage key: "annotool-data"
localStorage value: JSON.stringify({
  annotations: Annotation[],
  nextId: number
})
```


## 六、技术约束与限制

### 6.1 兼容性

- **浏览器**：Chrome / Firefox / Safari / Edge 最新版本
- **依赖**：零外部依赖，纯原生 JS
- **注入方式**：`<script src="../annotation-tool.js"></script>`

### 6.2 已知限制

| 限制 | 原因 | 应对方案 |
|------|------|---------|
| 区域标注不可编辑 | 无 DOM 选择器，无法重定位 | 提示用户重新框选 |
| 徽章在动态渲染页面可能不准 | 基于选择器查找，页面变化后失效 | 刷新页面重新加载标注 |
| 字数限制：旧文本截断 30 字符 | 面板显示宽度限制 | 导出中包含完整数据 |
| 上下文截断 600 字符 | 导出文件可读性 | 关键元素标记 data-target |
| iframe 内需每页单独注入 | 事件无法穿透 iframe 边界 | 在 iframe 加载的每个内容页的 `</body>` 前加 `<script src="annotation-tool.js">` |

### 6.3 安全性

所有用户输入内容经过 `escapeHtml()` 转义后插入 DOM，防止 XSS 注入。
